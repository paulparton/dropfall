# Phase 6 Plan 1: Online Multiplayer Input Routing Fix

## Objective

Fix the bug where Player 2 (client) joining an online game cannot control their blue ball because their inputs (arrow keys) were incorrectly mapped to control Player 1's red ball instead.

## Context

User requirement: "everyone uses Player 1 controls" (WASD) regardless of slot.

## Tasks

### Task 1: Fix resetOnlineEntities() in main.js

**Type:** auto  
**Files:** src/main.js

**Behavior (The Bug):**
- In online mode, when mySlot === 2 (client), line 257 creates player2 with `getPlayer2InputUnified`
- `getPlayer2InputUnified` reads `settings.controls.p2` (arrow keys)
- This causes client inputs to control player1 instead of their own player2

**Implementation (The Fix):**
- In resetOnlineEntities(), when mySlot === 2:
  - player1 gets `opponentInput` (remote host's inputs)
  - player2 gets `getPlayer1InputUnified` (local WASD controls) ← CHANGED from getPlayer2InputUnified
- This ensures the LOCAL player always uses WASD regardless of slot

### Task 2: Fix online input sync in main.js

**Type:** auto  
**Files:** src/main.js

**Behavior:**
- Line 1019 uses conditional to pick input getter based on playerSlot

**Implementation:**
- Always use `getPlayer1InputUnified()` for online sync since both players use WASD
