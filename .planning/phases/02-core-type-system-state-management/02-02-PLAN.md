**Exports (required):**

1. AudioPlaybackState: string literal union = 'stopped' | 'playing' | 'paused'

2. AudioLifecycle: string literal union = 'uninitialized' | 'initializing' | 'ready' | 'playing' | 'disposed'

3. SoundEffect: interface
   - id: string (e.g., 'sfx_collision', 'sfx_boost')
   - type: 'effect'
   - url?: string (optional, for future external sources)
   - duration: number (milliseconds)
   - volume: number (0-1)

4. MusicTrack: interface
   - id: string (e.g., 'music_game', 'music_menu')
   - type: 'music'
   - url?: string
   - duration: number
   - volume: number (0-1)
   - loop: boolean

5. AudioContext: interface
   - lifecycle: AudioLifecycle (tracks state: uninitialized → ready)
   - play(sound: SoundEffect | MusicTrack): Promise<void>
   - stop(soundId: string): Promise<void>
   - setVolume(soundId: string, volume: number): void
   - dispose(): Promise<void>
   - Comment: "Explicit lifecycle prevents race conditions in initialization"

6. PlaybackRequest: interface
   - soundId: string
   - volume?: number (defaults to sound's volume)
   - delay?: number (milliseconds before starting)

7. PlaybackEvent: interface
   - type: 'started' | 'stopped' | 'error'
   - soundId: string
   - error?: string (only if type='error')

8. Type guard function:
   - isAudioReady(context: AudioContext): boolean
   - Returns true if lifecycle === 'ready' or === 'playing'

9. Factory function:
   - createAudioContext(): AudioContext
   - Returns AudioContext with lifecycle='uninitialized'

**Rules:**
- NO implicit any types
- AudioLifecycle is the key to race condition prevention (must initialize before play)
- PlaybackRequest separates "what to play" from "how to play"

**Exports (required):**

   (Set to '1.0' for now; can bump to '1.1' if breaking changes)

2. ClientToServerMessage: discriminated union with type field:
   - JoinMessage: { type: 'join'; version: MessageVersion; playerId: string; gameMode: '1P'|'2P'; playerName: string }
   - StartRoundMessage: { type: 'start-round'; version: MessageVersion; roundNumber: number }
   - PlayerMoveMessage: { type: 'player-move'; version: MessageVersion; playerId: string; input: {x: number; y: number; boost: boolean} }
   - BoostMessage: { type: 'boost'; version: MessageVersion; playerId: string }
   - RoundEndMessage: { type: 'round-end'; version: MessageVersion; winnerId: string; score: {p1: number; p2: number} }
   - DisconnectMessage: { type: 'disconnect'; version: MessageVersion; playerId: string; reason: string }

3. ServerToClientMessage: discriminated union (mirror of ClientToServerMessage for now, can diverge later)
   - Same structure with additional: ackId?: string (for request-response pairs)

4. NetworkMessage: union = ClientToServerMessage | ServerToClientMessage with metadata:
   - type: string (inherited from the discriminant)
   - version: MessageVersion
   - timestamp: number (ms since epoch)
   - direction: 'client-to-server' | 'server-to-client'

5. NetworkPayload: interface (the raw bytes sent)
   - type: string
   - version: MessageVersion
   - data: unknown (parsed body)
   - raw: string (JSON serialized for debugging)

6. Type guard functions:
   - isNetworkMessage(obj: unknown): obj is NetworkMessage
   - Checks: version, type, direction fields
   - isVersionCompatible(version: string): boolean (checks if version in ['1.0'])

7. Message-specific guards (optional but helpful):
   - isPlayerMoveMessage(m: NetworkMessage): m is PlayerMoveMessage
   - isJoinMessage(m: NetworkMessage): m is JoinMessage

8. Constants:
   - export const CURRENT_PROTOCOL_VERSION: MessageVersion = '1.0'

**Rules:**
- NO implicit any types
- Use discriminant 'type' field for union discrimination
- Every message must include version for future compatibility
- timestamp added at serialization time (not in message source)
