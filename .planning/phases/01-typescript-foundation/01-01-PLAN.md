<task type="auto">
  <name>Create tsconfig.json with TypeScript 5.x strict mode</name>
  <files>tsconfig.json</files>
  <read_first>
    - CONTEXT.md (configuration decisions)
    - vite.config.js (existing build config to understand output)

  <action>
Create `tsconfig.json` at project root with the following configuration. This is the authoritative TypeScript configuration for the entire project.

```json
{
  "compilerOptions": {
    "target": "ES2020",
    "module": "ESNext",
    "lib": ["ES2020", "DOM", "DOM.Iterable"],
    
    "// Strict Mode - ALL ENABLED": "",
    "strict": true,
    "noImplicitAny": true,
    "noImplicitThis": true,
    "strictNullChecks": true,
    "strictFunctionTypes": true,
    "strictBindCallApply": true,
    "strictPropertyInitialization": true,
    "alwaysStrict": true,
    "noImplicitReturns": true,
    "noFallthroughCasesInSwitch": true,
    "noUncheckedIndexedAccess": true,
    
    "// Module Resolution": "",
    "moduleResolution": "bundler",
    "esModuleInterop": true,
    "allowSyntheticDefaultImports": true,
    "resolveJsonModule": true,
    
    "// Output & Declarations": "",
    "outDir": "./dist",
    "declaration": true,
    "declarationMap": true,
    "sourceMap": true,
    "removeComments": false,
    
    "// Emit": "",
    "emitDecoratorMetadata": false,
    "experimentalDecorators": false,
    
    "// Interop & Compatibility": "",
    "skipLibCheck": false,
    "allowJs": true,
    "checkJs": false,
    
    "// Diagnostics": "",
    "noEmit": true,
    "listFiles": false,
    "listFilesOnly": false
  },
  "include": ["src/**/*.ts", "server/**/*.ts", "tests/**/*.ts"],
  "exclude": ["node_modules", "dist", ".planning", ".git", ".venv"],
  "ts-node": {
    "compilerOptions": {
      "module": "commonjs"
    }
  }
}
```

**Rationale for each setting:**
- `strict: true` — Enables all strict checks with one flag (baseline)
- `skipLibCheck: false` — Verify type definitions are correct (catches library type issues)
- `allowJs: true` — Allow `.js` files to coexist during gradual migration
- `checkJs: false` — But don't enforce type checks on `.js` files yet (only `.ts`)
- `declaration: true` — Generate `.d.ts` for any exported modules (future library use)
- `sourceMap: true` — Enable debugging with original source in DevTools
- `noEmit: true` — Vite handles actual emission, tsconfig just type-checks
- `moduleResolution: bundler` — Vite's resolution strategy

  </action>

    <automated>test -f tsconfig.json && grep -q '"strict": true' tsconfig.json && grep -q '"allowJs": true' tsconfig.json && echo "✓ tsconfig.json valid"</automated>
  </verify>
  <done>
    - tsconfig.json exists at project root with strict mode enabled
    - All strict flags explicitly set (not just relying on "strict: true" shorthand)
    - allowJs: true permits .js/.ts coexistence
    - checkJs: false prevents false errors on existing .js files
    - File includes the exact settings from CONTEXT.md with no deviations
  </done>
