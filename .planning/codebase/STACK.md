# Technology Stack

**Analysis Date:** 2026-03-28

## Languages

**Primary:**
- JavaScript (ES6+) - Client game logic and server backend
- GLSL - Shader code for SDF ray-marching and post-processing

**Secondary:**
- HTML - Generated by Vite from index.html template
- CSS - Game styles in `src/style.css`

## Runtime

**Environment:**
- Node.js - Backend server (WebSocket multiplayer)
- Browser (Chromium, Firefox, Safari, Edge) - Client game
- Electron - Desktop packaging wrapper

**Package Manager:**
- npm
- Lockfile: `package-lock.json` (present)

## Frameworks & Core Libraries

**Rendering:**
- Three.js (v0.183.1) - WebGL 3D rendering engine
  - Used in `src/renderer.js` for scene, camera, lights, materials
  - Post-processing addons: EffectComposer, RenderPass, UnrealBloomPass, OutputPass

**Physics:**
- @dimforge/rapier3d-compat (v0.19.3) - WASM-based physics simulation
  - Used in `src/physics.js` and `src/sdf/physics.js` for rigid bodies, colliders, constraints
  - Desktop: 1/60 timestep, Mobile: 1/30 timestep

**State Management:**
- Zustand (v5.0.11) - Lightweight state store
  - Game state, player settings, UI visibility tracking
  - localStorage persistence via `src/store.js` defaultSettings

**Build & Dev:**
- Vite (v7.3.1) - Fast bundler and dev server
  - Config: `vite.config.js` with manual code splitting (three, physics chunks)
  - Dev server: port 5173, host mode for network access
  - Output: `dist/` with terser minification

**Testing:**
- Vitest (v4.1.0) - Unit test runner
  - Config: `vitest.config.js` with jsdom environment
  - Test files in `tests/` directory

**Networking:**
- ws (v8.18.0) - WebSocket server for multiplayer
  - Server: `server/server.js` (Node.js HTTP + WebSocket combined)
  - Client: `src/online.js` (OnlineManager class)

**Testing Support:**
- @testing-library/dom (v10.4.1) - DOM test utilities
- jsdom (v29.0.1) - Browser environment simulator

## Key Dependencies Summary

**Production:**
- `three` - 3D graphics (largest bundle)
- `@dimforge/rapier3d-compat` - Physics engine (separate chunk)
- `zustand` - State management (<1KB)
- `ws` - WebSocket communication

**Dev:**
- `vite` - Build orchestration
- `vitest` - Test runner
- `@testing-library/dom`, `jsdom` - Test environment

Total dependencies: 4 production, 4 dev

## Configuration Files

**Build:**
- `vite.config.js` - Bundler config with:
  - Manual chunks: `three`, `physics` separated
  - terser minification, source maps disabled
  - Output dir: `dist/`
  - Dev server: 0.0.0.0:5173 (network accessible)
  - Preload: three, zustand, @dimforge/rapier3d-compat

**Testing:**
- `vitest.config.js` - Test runner:
  - Environment: jsdom
  - Include pattern: `tests/**/*.test.js`
  - Coverage: text + HTML reporters

**Package:**
- `package.json` - NPM config with ES modules (`"type": "module"`)

## Scripts

```bash
npm run dev           # Vite with network host (0.0.0.0:5173)
npm run dev:local    # Vite localhost only (5173)
npm run build        # Production bundle to dist/
npm run preview      # Serve dist/ preview
npm run server       # Node.js multiplayer backend (port 3000)
npm run server:dev   # Server with --watch auto-restart
npm test             # Run all tests once
npm test:watch      # Run tests in watch mode
```

## Platform Requirements

**Development:**
- Node.js (LTS or latest)
- npm or compatible package manager
- Modern browser (ES6+, WebGL support)

**Production:**
- Web: Static hosting + Node.js server (for multiplayer)
- Desktop: Electron runtime
- Browser: WebGL, WebSocket support required

## Browser APIs

**Graphics:**
- WebGL (via Three.js WebGLRenderer)
- Canvas 2D (fallback, if used)
- requestAnimationFrame for game loop

**Input:**
- Keyboard (KeyboardEvent)
- Gamepad API in `src/input.js`
- Touch events (mobile detection logic in `src/renderer.js`)

**Storage:**
- localStorage for game settings, player names, preferences
  - Keys: `dropfall_settings`, `dropfall_gamemode`, `dropfall_difficulty`, `dropfall_p1name`

**Networking:**
- WebSocket API (ws:// or wss://)

**Audio:**
- Web Audio API (synthesized audio, no external files)
  - Managed in `src/audio.js`

## Mobile Considerations

**Optimizations:**
- User-Agent detection + viewport width check (<768px = mobile)
- Pixel ratio: 1.0 mobile, up to 2x desktop
- Antialias: disabled mobile, enabled desktop
- Shadow map: 512x512 mobile, 1024x1024 desktop
- Physics timestep: 30fps mobile, 60fps desktop

## Deployment

**Web:**
- Static hosting (dist/ directory) + Node.js server for multiplayer
- Vite/Vercel/Netlify compatible

**Desktop:**
- Electron packaging
- Preload script: `electron/preload.js` (currently minimal)
